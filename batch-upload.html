Batch Upload Guide

Issue multiple certificates at once using a CSV file.

Overview

The batch upload feature lets you issue up to 25 certificates per batch by uploading a CSV file. If your CSV has more than 25 rows, the platform will offer to split it into multiple batches automatically.

CSV Format

Your CSV file must include a header row. The following columns are supported:

ColumnRequiredDescription
name Yes The name of the certified company or individual.
competency Yes The qualification or certification type (e.g. "First Aid", "ISO 9001").
expiry Yes The certificate expiry date. See Date Format below.
companyEmail No Recipient's email address for certificate delivery.
imageURI No A direct URL to a certificate image. See imageURI below.

Example CSV

name,companyEmail,competency,expiry,imageURI
Acme Ltd,admin@acme.example,First Aid,2030-12-31,
Beta Corp,info@beta.example,ISO 9001,2028-06-15,https://example.com/images/beta-cert.png
Gamma Inc,,Fire Safety,12/31/2027,

Date Format (expiry column)

The expiry column accepts two date formats:

FormatPatternExample
ISO (recommended) YYYY-MM-DD 2026-12-31
US MM/DD/YYYY 12/31/2026
Tip We recommend using ISO format (YYYY-MM-DD) to avoid ambiguity between day and month, especially if your data comes from different regions.
Common mistake Dates like 01/02/2026 are interpreted as January 2nd (US format), not February 1st. Use ISO format if this could cause confusion.

imageURI Column

The imageURI column is optional. When provided, it must be a direct URL to an image file.

What counts as a direct image URL?

  • The URL should point directly to an image file (typically ending in .png, .jpg, .jpeg, .gif, or .webp).
  • When you open the URL in a browser, you should see only the image — not a webpage with the image embedded in it.
CorrectIncorrect
https://example.com/images/cert.png https://example.com/certificates/view/123
https://cdn.example.com/logo.jpg https://drive.google.com/file/d/abc123/view
Important A webpage link (e.g. a Google Drive sharing link or a page that contains an image) will not work. The URL must point directly to the image file so it can be downloaded and embedded in the certificate.

Download the Template

On the Certificates page, click the Download template button in the "Batch issue (CSV)" section to get a pre-formatted CSV with the correct headers and an example row.

Uploading Your CSV

  1. Go to Certificates > Create and scroll to the "Batch issue (CSV)" section.
  2. Click the file input and select your CSV file.
  3. The platform will validate your CSV and show a preview table with each row's status.
  4. Rows marked OK are ready to mint. Rows with errors will show the issue (e.g. missing required field).
  5. Click Mint batch to issue all valid certificates.

Large CSV Files (more than 25 rows)

If your CSV contains more than 25 rows, the platform will detect this and offer you three options:

  • Split into batches — Automatically split the CSV into chunks of 25 and mint them sequentially.
  • Use first 25 only — Only process the first 25 rows.
  • Cancel — Go back and modify your CSV.

Troubleshooting

"CSV must include headers: name, competency, expiry"

Your CSV is missing one or more required columns. Make sure the first row contains exactly these header names (case-sensitive).

Rows showing errors in preview

Check that each row has values for name, competency, and expiry. Dates must be in one of the accepted formats.

imageURI not loading

Ensure the URL is a direct link to the image file. Try opening the URL in a new browser tab — if you see the image by itself (not embedded in a webpage), the URL is correct.
